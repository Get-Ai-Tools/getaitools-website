What it does

Extracts text from any PDF and sends it to Claude AI for summarization. Works entirely in your browser - your document and API key stay on your device. For large files, content is automatically truncated to fit within context limits.

Features

  • Three output modes: paragraph summary, bullet points, or key takeaways
  • Model switcher - use Haiku for speed/cost, Opus for complex documents
  • API key saved locally in your browser - never hits our servers
  • Python script available for batch processing and automation
  • Costs fractions of a cent per PDF with Haiku

Quick start (Python)

git clone https://github.com/Get-Ai-Tools/pdf-summarizer
cd pdf-summarizer
pip install -r requirements.txt

python summarizer.py --api-key sk-ant-... --pdf report.pdf
python summarizer.py --api-key sk-ant-... --folder ./pdfs --out ./summaries

Frequently asked questions

Is my API key safe?

Yes. Your key is saved in your browser's localStorage only. When you hit Summarize, your browser sends the key and document text to a thin Cloudflare proxy function that immediately forwards it to Anthropic - nothing is logged or stored. The proxy source code is public.

Does it work on scanned PDFs?

It works best on PDFs with selectable text (research papers, reports, ebooks). Scanned image PDFs have no extractable text - you'd need an OCR tool first.

How much does it cost?

Using Claude Haiku (the default), a typical 10-page PDF costs around $0.001-$0.005. You're billed directly by Anthropic - we never touch your payment details.

What's the file size limit?

No hard limit on file size - but only the first ~80,000 characters of text are processed to stay within model context limits. For most documents this is plenty.
